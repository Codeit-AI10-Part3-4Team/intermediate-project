# notebooks/test/

## 역할
`notebooks/` 구현과 연관된 **단위/통합 테스트 노트북**을 보관합니다.
외부 API 연결 확인, 라이브러리 동작 검증 등 **실험 전에 환경이 제대로 갖춰졌는지 점검**하는 노트북이 여기에 위치합니다.

- `src/`에 대한 정식 테스트는 `tests/`(pytest)에서 관리합니다. 이 폴더는 **노트북 환경에서의 점검**이 목적이며 역할이 다릅니다.
- 여기서 검증된 호출 로직은 프로덕션에 필요할 경우 `src/rag_core/`의 대응 모듈로 함수화해 이전하고, 테스트는 `tests/`로 옮깁니다.

## 노트북 목록

### `OpenAIAPITest.ipynb` — OpenAI API 연결 스모크 테스트
OpenAI API 키와 네트워크 연결이 정상 동작하는지 최소 호출로 확인합니다.

- 동작: `OpenAI()` 클라이언트로 `gpt-5-nano` 모델에 `"ping"`을 보내고 응답을 출력 → 키/엔드포인트/SDK가 살아있는지 확인.
- 사전조건
  - `pip install openai`
  - 환경 변수 `OPENAI_API_KEY` 설정 (코드/노트북에 키를 평문으로 적지 마세요). `OpenAI()`는 이 환경 변수를 자동으로 읽습니다.
- 목적: 실제 RAG 실험(`notebooks/llm/` 등) 전에 **LLM 호출 환경을 사전 점검**하는 용도. 비즈니스 로직이 아니므로 `src/`로 이전 대상은 아닙니다.
