## Part 1 — 가상환경 생성

### 개요

팀원 간 패키지 충돌 방지를 위해 **본인 전용 가상환경**을 생성합니다.
이후 모든 작업은 이 환경 위에서 진행합니다.

---

### Step 1. JupyterHub 로그인

브라우저에서 사전 조건의 URL로 접속한 후, 본인 ID로 로그인합니다.

---

### Step 2. 가상환경 생성 및 커널 등록

`{이름}` 부분에 본인 ID 등 **팀 내에서 구분 가능한 이름**을 사용하세요.

**터미널에서 실행**

```bash
python3 -m venv ~/{이름}
~/{이름}/bin/python -m ipykernel install --user --name {이름} --display-name "Python ({이름})"
```

**Notebook에서 실행**

```python
!python3 -m venv ~/{이름}
!~/{이름}/bin/python -m ipykernel install --user --name {이름} --display-name "Python ({이름})"
```

> **참고:** 설치가 완료되면 홈 디렉터리에 `~/{이름}/` 폴더가 생성됩니다.

---

### Step 3. 커널 전환

환경 생성 후 반드시 커널을 새 환경으로 전환해야 합니다.

**터미널에서 전환**

```bash
source ~/{이름}/bin/activate
```

**Notebook에서 전환**

1. 화면 **우측 상단**의 현재 커널 이름(예: `Python 3 (ipykernel)`)을 클릭합니다.
2. `Select Kernel` 팝업의 드롭다운에서 **`Python ({이름})`** 을 선택합니다.
3. `Always start the preferred kernel`을 체크한 후 **`Select`** 버튼을 누릅니다.

> **커널이 목록에 보이지 않을 경우:**
> 상단 메뉴 `Kernel → Restart Kernel and Clear Outputs`를 실행한 후 재시도하세요.

## Part 2 — 프로젝트 패키지 설치

### 개요

`pip install -e .`는 프로젝트 소스 코드를 **복사하지 않고 원본 경로를 직접 참조**하도록 등록합니다.
소스 파일을 수정하면 재설치 없이 변경 사항이 즉시 반영됩니다.

---

### Step 1. 커널 확인

Part 1에서 생성한 가상환경 커널이 활성화되어 있는지 확인합니다.

---

### Step 2. 프로젝트 패키지 설치

> **⚠️ 주의:** 반드시 `pyproject.toml`이 있는 **프로젝트 루트 디렉터리**에서 실행하세요.
> 혹은 저장소 루트에 있는 SetProjectPakage.ipynb를 실행하세요.

**터미널에서 실행**

```bash
cd /path/to/project-root  # pyproject.toml이 있는 경로
pip install -e .
```

**Notebook에서 실행**

```python
%pip install -e .
```

## Part 3 — 패키지 설치 확인

### 개요

설치가 Editable 모드(-e)로 정상적으로 완료되었는지 확인합니다.

---

### Step 1. 확인 스크립트 실행

아래 코드를 **Notebook 셀** 또는 별도 `.py` 파일로 실행합니다.  

```python
import importlib.util
import logging
from pathlib import Path

logger = logging.getLogger(__name__)
logging.basicConfig(level=logging.INFO)


def check_editable_installation(package_name: str) -> bool:
    """
    프로젝트가 Editable(-e) 모드로 설치되었는지 검증합니다.
    
    Args:
        package_name: pyproject.toml의 project.name 값
        
    Returns:
        설치가 정상이면 True, 아니면 False
    """
    try:
        spec = importlib.util.find_spec(package_name)
        if spec is None:
            raise ModuleNotFoundError(
                f"'{package_name}' 패키지를 찾을 수 없습니다."
            )

        package_origin = spec.origin
        logger.info(f"패키지 로드 성공 — 연결 경로: {package_origin}")

        # Editable 모드 여부: 소스 원본 경로를 직접 참조하는지 확인
        # Notebook 환경에서는 __file__ 이 정의되지 않으므로 별도 처리
        try:
            current_dir = Path(__file__).resolve().parent
            if str(current_dir) in str(package_origin):
                logger.info("✓ Editable(-e) 모드로 소스 원본과 실시간 연결되어 있습니다.")
        except NameError:
            # Jupyter Notebook 환경에서는 __file__ 미정의 — 경로 검증 생략
            logger.info("✓ Notebook 환경에서는 경로 검증을 건너뜁니다.")

        return True

    except ModuleNotFoundError as e:
        logger.error(f"패키지 미설치: {e}")
        print("\n[조치] 프로젝트 루트(pyproject.toml 위치)에서 아래 명령을 실행하세요:")
        print("  pip install -e .")
        return False

    except Exception as e:
        logger.error(f"예상치 못한 오류: {e}")
        return False


# ── 스크립트(.py)로 직접 실행할 경우 ──────────────────────────────────
if __name__ == "__main__":
    # pyproject.toml의 project.name 값을 입력하세요
    check_editable_installation("intermediate-project")

# ── Notebook 셀에서 실행할 경우 ────────────────────────────────────────
# check_editable_installation("intermediate-project")
```

---

### Step 2. 정상 설치 시 출력 예시

```
INFO:__main__:패키지 로드 성공 — 연결 경로: /home/{이름}/project/src/intermediate_project/__init__.py
INFO:__main__:✓ Editable(-e) 모드로 소스 원본과 실시간 연결되어 있습니다.
```

---

### Step 3. 오류 발생 시 체크리스트

| 증상 | 원인 | 조치 |
|------|------|------|
| `ModuleNotFoundError` | `pip install -e .` 미실행 | Part 2 Step 2 재실행 |
| 패키지는 보이지만 변경 사항이 미반영 | 다른 환경의 커널 사용 중 | Part 1 Step 3 커널 전환 재확인 |
| 커널 목록에 환경이 없음 | ipykernel 등록 누락 | Part 1 Step 2 재실행 후 커널 재시작 |
