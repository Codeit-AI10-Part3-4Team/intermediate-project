# eval/

## 역할
RAG 파이프라인의 성능을 측정하기 위한 평가 자산(골든 데이터셋, 평가 기준, 평가 스크립트)을 보관합니다.
`src/`의 코드와 라이프사이클이 다르므로(데이터셋은 버전이 바뀌고, 코드는 리팩토링됨) 별도 최상위 디렉토리로 분리되어 있습니다.

## 담당
PM

## 구조 가이드
```
eval/
├── golden_dataset/      # 질의-정답 쌍 50개 (버전 관리 대상)
├── metrics.py            # 평가 지표 계산 함수 (예: recall@k, faithfulness 등)
└── eval_criteria.md       # 평가 기준 문서
```

## 산출물 연계
- `notebooks/eval/`에서 구축 과정을 기록하고, 확정된 데이터셋만 `golden_dataset/`로 이전합니다.
- `src/rag_core/` 전체 파이프라인(또는 단계별)의 출력을 `golden_dataset/`의 정답과 비교해 `metrics.py`로 점수를 계산합니다.

## 코딩 에이전트 참고
- `golden_dataset/`의 파일은 평가 기준 자체이므로 코드 리팩토링과 무관하게 신중히 변경합니다. 변경 시 변경 이유를 커밋 메시지에 명시하세요.
- `metrics.py`는 `src/rag_core/`의 특정 구현에 의존하지 않고, 입력(예측 결과 리스트)과 정답만 받아 점수를 계산하는 순수 함수로 작성하세요. 이렇게 해야 파이프라인 구현이 바뀌어도 평가 코드를 재사용할 수 있습니다.
