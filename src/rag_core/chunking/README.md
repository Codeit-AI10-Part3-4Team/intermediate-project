# src/rag_core/chunking/

## 역할
`parsing/`에서 추출된 텍스트를 검색·임베딩에 적합한 단위(청크)로 분할하는 모듈입니다.

## 담당
희원님

## 입력/출력
- 입력: `parsing/`이 반환한 정규화된 텍스트/구조화 객체
- 출력: 청크 리스트(각 청크는 원본 위치 메타데이터 포함 — 출처 추적용)

## 산출물 연계
- 다음 단계인 `embedding/`(희원님)이 이 모듈의 출력을 입력으로 받습니다.
- `notebooks/retrieval/`에서 비교 실험한 청킹 전략 중 채택된 것만 이곳에 구현됩니다.

## 코딩 에이전트 참고
- 청킹 전략(고정 길이, 문단 기반, 문장 기반 등)이 여러 개 실험될 수 있으므로, 전략별로 함수를 분리하고 공통 인터페이스(예: `chunk(text, **params) -> list[Chunk]`)를 유지하세요.
- 청크 크기/overlap 등 하이퍼파라미터는 함수 인자 또는 설정 파일로 분리하고 하드코딩하지 않습니다.
- 임베딩/Retrieval 로직을 여기에 작성하지 마세요 (`embedding/`, `retrieval/` 담당 영역).
