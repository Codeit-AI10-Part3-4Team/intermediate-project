# src/rag_core/retrieval/

## 역할
사용자 질의에 대해 `embedding/`에서 생성된 벡터 중 관련 청크를 검색하는 모듈입니다.

## 담당
희원님

## 입력/출력
- 입력: 사용자 질의 텍스트, 검색 대상 임베딩 인덱스(또는 벡터 컬렉션)
- 출력: 관련도 순으로 정렬된 청크 리스트(+ 유사도 점수)

## 산출물 연계
- 다음 단계인 `prompts/`(지우님)가 이 모듈의 출력을 context로 받아 LLM 프롬프트를 구성합니다.
- "Retrieval Pipeline 초안"이 이 모듈과 `chunking/`, `embedding/`을 통합한 첫 결과물입니다.

## 코딩 에이전트 참고
- Baseline은 brute-force cosine similarity로 시작하되, 추후 벡터 인덱스(예: FAISS, HNSW)로 교체 가능하도록 검색 인터페이스를 추상화하세요 (예: `Retriever` 인터페이스/베이스 클래스 + `BruteForceRetriever` 구현체).
- 검색 결과가 비어있는 경우(질의와 관련된 청크가 없는 경우)에 대한 처리를 명시적으로 포함합니다.
- top-k 값은 하드코딩하지 말고 함수 인자로 노출하세요.
- 임베딩 생성/프롬프트 구성 로직을 여기에 작성하지 마세요.
