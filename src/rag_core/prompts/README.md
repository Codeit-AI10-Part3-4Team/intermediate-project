# src/rag_core/prompts/

## 역할
`retrieval/`의 검색 결과(context)와 사용자 질의를 조합해 LLM에 전달할 프롬프트를 구성하는 모듈입니다.

## 담당
지우님

## 입력/출력
- 입력: 사용자 질의, `retrieval/`이 반환한 관련 청크 리스트
- 출력: LLM에 전달할 최종 프롬프트 문자열(또는 메시지 리스트)

## 구조 가이드
```
prompts/
├── templates/
│   └── prompt_template_v1.txt   # 프롬프트 템플릿 원본
└── builder.py                    # 템플릿 + context를 조합하는 로직
```

## 산출물 연계
- 다음 단계인 `llm/`(지우님)이 이 모듈의 출력을 입력으로 받아 LLM을 호출합니다.
- `notebooks/llm/`에서 실험한 프롬프트 중 채택된 버전이 `templates/`에 버전 관리됩니다(`v1`, `v2` 등 명시적 버전 네이밍).

## 코딩 에이전트 참고
- 프롬프트 텍스트는 Python 코드에 직접 문자열로 작성하지 말고 `templates/` 하위 파일로 분리합니다. 새 버전을 만들 때는 기존 파일을 덮어쓰지 말고 `_v2`, `_v3`처럼 새 파일을 추가해 이전 버전과 비교 가능하게 합니다.
- 검색 결과 청크가 너무 많아 컨텍스트 길이를 초과할 가능성에 대한 처리(잘라내기 또는 우선순위 선택)를 포함합니다.
- LLM 호출 로직을 여기에 작성하지 마세요 (`llm/` 담당 영역).
