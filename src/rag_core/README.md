# src/rag_core/

## 역할
문서 업로드 후 처리되는 RAG(Retrieval-Augmented Generation) 파이프라인의 핵심 로직을 담는 패키지입니다.
FastAPI 등 서빙 레이어와 독립적으로 동작해야 하며, 노트북(`notebooks/`)에서 검증이 끝난 코드만 이곳으로 이전됩니다.

## 파이프라인 순서 및 하위 패키지
```
parsing → chunking → embedding → retrieval → prompts → llm
```
- `parsing/` — PDF/HWP 파일에서 텍스트/구조 추출 (담당: 유빈님 조사 → 이전)
- `chunking/` — 추출된 텍스트를 청크 단위로 분할 (담당: 희원님)
- `embedding/` — 청크를 벡터로 변환 (담당: 희원님)
- `retrieval/` — 질의에 대한 관련 청크 검색 (담당: 희원님)
- `prompts/` — 검색 결과를 LLM 입력 프롬프트로 구성 (담당: 지우님)
- `llm/` — LLM 호출 및 응답 처리 (담당: 지우님)

## 의존성 규칙(중요)
- 이 패키지는 `src/api`를 import하지 않습니다.
- 패키지 간에는 파이프라인 순서를 따르는 단방향 의존만 허용합니다 (예: `retrieval`이 `embedding`을 사용하는 것은 가능, 역방향은 지양).
- 각 서브패키지는 외부에서 호출할 함수를 `__init__.py`에서 명시적으로 export합니다.
