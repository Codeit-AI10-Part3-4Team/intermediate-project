# src/rag_core/parsing/

## 역할
PDF/HWP 등 원본 문서 파일에서 텍스트와 구조(섹션, 표, 목록 등)를 추출하는 모듈입니다.

## 담당
유빈님

## 입력/출력
- 입력: 원본 문서 파일 경로 또는 바이트 스트림 (`data/` 또는 업로드 API로부터 전달)
- 출력: 정규화된 텍스트(또는 구조화된 객체 — 섹션 제목, 본문, 표 분리)

## 산출물 연계
- 다음 단계인 `chunking/`(희원님)이 이 모듈의 출력을 입력으로 받습니다.
- `notebooks/parsing/`에서 검증된 파싱 로직만 이곳에 함수화되어 들어옵니다.

## 코딩 에이전트 참고
- PDF와 HWP는 파서 라이브러리가 다르므로 `pdf_parser.py`, `hwp_parser.py`로 파일을 분리하고, 공통 인터페이스(예: `parse(file_path) -> ParsedDocument`)를 맞춥니다.
- 파싱 실패(손상된 파일, 지원하지 않는 포맷)에 대한 예외 처리를 반드시 포함합니다. 예외를 삼키지 말고 명확한 예외 타입으로 raise하여 `api/` 레이어에서 적절한 HTTP 에러로 변환할 수 있게 합니다.
- 청킹/임베딩 로직을 여기에 작성하지 마세요 (`chunking/` 담당 영역).
