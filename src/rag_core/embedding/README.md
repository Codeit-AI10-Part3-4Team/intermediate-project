# src/rag_core/embedding/

## 역할
`chunking/`에서 분할된 청크를 벡터(임베딩)로 변환하는 모듈입니다.

## 담당
희원님

## 입력/출력
- 입력: 청크 리스트(`chunking/`의 출력)
- 출력: 청크별 임베딩 벡터(float32 배열) + 원본 청크 메타데이터

## 산출물 연계
- 다음 단계인 `retrieval/`(희원님)이 이 모듈의 출력을 입력으로 받습니다.
- `notebooks/retrieval/`에서 비교한 임베딩 모델 후보 중 채택된 모델만 이곳에 구현됩니다.

## 코딩 에이전트 참고
- 임베딩 모델 교체 가능성을 고려해 모델 종류를 인자/설정으로 분리하세요 (예: `embed(texts, model_name) -> np.ndarray`). 특정 모델 SDK에 강하게 결합된 코드는 지양합니다.
- 임베딩 결과(벡터)는 git에 직접 커밋하지 않습니다 (`data/` 또는 별도 저장소에 `.gitignore` 처리). 재현 가능한 생성 스크립트만 이곳에 둡니다.
- 배치 처리 시 메모리 사용량을 고려해 청크 수가 많을 경우 배치 단위 처리를 구현하세요.
- 파싱/청킹/Retrieval 로직을 여기에 작성하지 마세요.
